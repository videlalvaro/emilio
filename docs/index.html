One Primitive: Running a Transformer and Game of Life on Apple’s Neural Engine via Convolution

Both Conway’s Game of Life and Qwen 2.5‑0.5B compile to the same instruction: convolution on the ANE’s 16‑core engine.

Alvaro Videla · April 2026 · Source code

171 tok/s Decode throughput on Apple M4 Max — Qwen 2.5‑0.5B running entirely on the Neural Engine via Conv2d(1×1).

The Idea

Apple’s Neural Engine is a hardware convolution accelerator present in every Apple Silicon chip. Unlike the GPU—which runs arbitrary parallel kernels—the ANE achieves peak throughput on a narrow set of operations: small-kernel convolutions, element-wise arithmetic, and activation functions.

This project demonstrates that convolution alone is a sufficient primitive to implement two seemingly different computations on the same silicon:

  1. Conway’s Game of Life — a cellular automaton where each generation is a Conv2d(3×3) counting neighbors, followed by ReLU-based rule evaluation.
  2. Transformer inference — a full Qwen 2.5‑0.5B language model where every matrix multiplication is a Conv2d(1×1) operation.

The connection is not metaphorical. Both computations compile to the same ANE instruction stream: convolution kernels dispatched to the same 16-core engine.

Game of Life as Convolution

Conway’s Game of Life operates on a binary grid. The rules depend on alive-neighbor count: birth (exactly 3), survival (2 or 3), death (otherwise). We encode one generation as a single convolution with a carefully chosen kernel:

K = [[1, 1, 1],
     [1, 9, 1],
     [1, 1, 1]]

The center weight 9 encodes the cell’s own state into the output, separating dead cells (sum 0–8) from alive cells (sum 9–17) into non-overlapping integer ranges. This enables branchless rule evaluation via ReLU:

eq(s, k) = relu(1 − (relu(s − k) + relu(k − s)))

alive′ = clamp(eq(s, 3) + eq(s, 11) + eq(s, 12), 0, 1)

Where s=3 is birth, s=11 is survival with 2 neighbors, and s=12 is survival with 3. The entire generation is one conv + a few ReLU ops — natively accelerated on the ANE.

Transformer as Convolution

A transformer processes one token at a time through a stack of identical layers. Each layer performs five steps: normalize (RMSNorm), project to Q/K/V, attend, project output, and feed-forward network. After 24 layers, a final projection maps the 896-dimensional vector to 151,936 vocabulary logits.

The key observation: all the expensive operations are matrix–vector products y = Wx. The nonlinearities (SiLU, softmax, RMSNorm) are element-wise and cheap.

Why Conv2d(1×1) = Matrix Multiply

When spatial dimensions are 1×1, a convolution degenerates to a weighted sum across input channels, repeated for each output channel. That is exactly matrix–vector multiplication:

yi = ∑j Wij · xj

We reshape the transformer’s hidden state from shape (896,) to (1, 896, 1, 1) — 896 “channels” of a single 1×1 “image”. A Conv2d(896, 1152, kernel_size=1) then produces the QKV projection. No approximation — the operations are mathematically identical.

169 matrix multiplications per token (7 per layer × 24 layers + LM head), all dispatched to the ANE’s convolution engine.

CoreML’s MLState API (macOS 15) keeps the KV cache on-device: the model writes to it in-place via registered state buffers. Pack time dropped from 23% to 0.00ms.

The Punchline

The Game of Life uses Conv2d(3×3). The transformer uses Conv2d(1×1). Both run on the same 16-core convolution engine. Same silicon, same primitive, two completely different computations.

5.67ms Per-token latency. Within 2× of the DRAM bandwidth floor (4.7ms theoretical for 472 MB int8 weights at 100 GB/s).
The remaining gap is scheduling overhead and element-wise ops between convolutions.

Technical Details

Model

  • Qwen 2.5-0.5B-Instruct — 24 layers, d=896, 14 query heads, 2 KV heads
  • Int8 block-wise quantization (block size 32) — 472 MB
  • Grouped Query Attention batched to 96 matmuls (down from 672)

Stack

  • Model conversion: Python + coremltools 9.0 → CoreML .mlpackage
  • Inference: Swift + CoreML + MLState (macOS 15+)
  • Hardware: Apple M4 Max

What Runs Where

  • ANE convolution engine (16 cores): All 169 matrix multiplications per token
  • ANE element-wise ALUs: RMSNorm, RoPE, softmax, SiLU
  • CPU: Tokenization, argmax, embedding lookup
