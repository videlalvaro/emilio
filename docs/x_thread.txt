── POST 2 (GOL encoding) ─────────────────────────────────────────

The Game of Life encodes one generation as a single 3×3 convolution:

K = [[1, 1, 1],
     [1, 9, 1],
     [1, 1, 1]]

The center weight 9 separates dead (0–8) from alive (9–17) into non-overlapping ranges. ReLU-based rule evaluation finishes it — branchless, fully on the ANE.

2.3 billion cell-generations/sec on a 1024×1024 grid.

[ATTACH: gol_evolution.gif]


── POST 3 (the key insight) ───────────────────────────────────────

A transformer is mostly matrix–vector products: y = Wx

When spatial dimensions are 1×1, a convolution IS a matrix–vector product:

y_i = Σ_j W_ij · x_j

Reshape the hidden state from (896,) to (1, 896, 1, 1) — 896 "channels" of a single-pixel "image." Conv2d does the rest. No approximation.


── POST 4 (mapping) ──────────────────────────────────────────────

Qwen 2.5-0.5B mapped entirely to Conv2d(1×1):

• QKV projection: Conv(896, 1152, 1)
• Output proj: Conv(896, 896, 1)
• Gate+Up: Conv(896, 9728, 1)
• Down proj: Conv(4864, 896, 1)
• LM head: Conv(896, 151936, 1)

169 matmuls per token → 169 convolutions per token. All on the ANE.

── POST 5 (optimization journey) ─────────────────────────────────

The optimization journey: 9 → 171 tok/s (19×)

• Baseline: 9 tok/s (naïve host code)
• Book-driven fixes: 90 tok/s (fused KV scatter, batched GQA: 672→96 attn matmuls)
• Pre-allocated buffers: ANE utilization 99.5%
• Fixed-shape model: skip dynamic padding
• Stateful KV (CoreML MLState): 171 tok/s — KV cache lives on ANE, zero host copy

[ATTACH: dataflow.gif]


── POST 6 (stateful KV detail) ────────────────────────────────────

The biggest win: CoreML's MLState API (macOS 15).

Before: 23% of time packing KV tensors into MLMultiArray. Another ~40% waiting for ANE to read them.

After: KV cache stays on-device. The model writes to it in-place. Pack time: 0.00ms.

5.67ms per token. Within 2× of the DRAM bandwidth floor (4.7ms theoretical for 472 MB int8 @ 100 GB/s).


── POST 7 (the punchline) ────────────────────────────────────────

The punchline:

GOL: Conv2d(3×3) — spatial convolution
Transformer: Conv2d(1×1) — pointwise convolution

Same ANE. Same 16-core convolution engine. Two completely different computations from one primitive.
